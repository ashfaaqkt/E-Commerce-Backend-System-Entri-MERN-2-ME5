E-Commerce Platform Deployment & Setup Instructions
---------------------------------------------------
This guide walks you through connecting MongoDB, integrating optional AI features, and hosting the application on Vercel and Render.

STEP 1: Connecting MongoDB
--------------------------
1. Go to https://account.mongodb.com/ and sign in or create an account for MongoDB Atlas.
2. Create a new Cluster (the free shared tier M0 is sufficient).
3. Once the cluster is provisioned, click "Connect".
4. Add your current IP address to the Network Access list.
5. Create a database user and generate a secure password.
6. Choose "Connect your application" (Node.js driver) and copy the connection string.
7. Open the `ecommerce-backend/.env` file.
8. Replace the `MONGO_URI` value with your connection string. 
   Make sure to replace `<password>` with the password you generated in step 5.
   Example: MONGO_URI=mongodb+srv://admin:mysecurepassword@cluster0.abcde.mongodb.net/ecommerce-db?retryWrites=true&w=majority

STEP 2: Optional AI Features API (Free Tier)
--------------------------------------------
If you want to integrate a real AI Recommendation API instead of the simulation (in `analyticsController.js`):
1. **Hugging Face Inference API (Free):** 
   - Sign up at https://huggingface.co/
   - Go to Settings -> Access Tokens and generate a Read token.
   - Use axios in your backend to send product titles to a similarity model like `sentence-transformers/all-MiniLM-L6-v2` to find related products.
2. **OpenAI API (Tiered):**
   - Alternatively, you can use OpenAI but it requires a paid tier. Free-tier Groq API (https://console.groq.com/) is a lightning-fast option for text completion if you want LLM features.
3. Once you choose a provider, add their API Key to `ecommerce-backend/.env` (e.g., `HF_API_KEY=your_token_here`).
4. Update `ecommerce-backend/controllers/analyticsController.js` to replace the `Math.random` logic with an Axios request to the AI provider.

STEP 3: Hosting Instructions (Backend on Render, Frontend on Vercel)
--------------------------------------------------------------------
It is highly recommended to host the Node.js Backend on Render and the React Frontend on Vercel. Vercel isn't designed for long-running Express server instances smoothly (it relies on serverless functions usually).

### Deploying the Backend on Render
1. Push your repository code to GitHub:
   git add .
   git commit -m "Backend and Frontend Setup"
   git push -u origin main
2. Sign in to https://render.com/ and create a new "Web Service".
3. Connect your GitHub account and select your `E-Commerce-Backend-System-Entri-MERN-2-ME5` repository.
4. Settings:
   - Root Directory: `ecommerce-backend`
   - Build Command: `npm install`
   - Start Command: `node index.js`
5. Environment Variables:
   Add your `MONGO_URI`, `JWT_SECRET`, and `PORT` (usually they assign one, but good to add).
6. Click "Create Web Service". Wait for the build and grab the URL (e.g., `https://ecommerce-backend-xyz.onrender.com`).

### Deploying the Frontend on Vercel
1. Sign in to https://vercel.com/ and click "Add New... -> Project".
2. Import the `E-Commerce-Backend-System-Entri-MERN-2-ME5` GitHub repository.
3. Configure Project Settings:
   - Framework Preset: Vite
   - Root Directory: `ecommerce-frontend`
4. Environment Variables:
   - Name: `VITE_API_URL`
   - Value: `<Your Render Backend URL>/api` (e.g., `https://ecommerce-backend-xyz.onrender.com/api`)
5. Click "Deploy". 
6. Vercel will build and assign you a live domain for your beautiful blue-gradient frontend!

Congratulations! Your End-to-End MERN Platform is live.
